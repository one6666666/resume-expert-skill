# AI 简历专家 | 2026 AI 时代简历生成与优化

## 概述

本 Skill 提供从零生成简历、JD 驱动优化、ATS 兼容检测到面试准备的全流程能力。一次输入，直接输出可投递的专业简历（HTML + DOCX）。

**本 Skill 已适配 2026 年 AI 时代的简历标准**：不仅要写传统开发技能，更要体现 AI 应用开发、AI 编程工具使用、运维监控、SEO/GEO、以及独立交付产品的能力。

**本 Skill 同时覆盖国内主流招聘平台（BOSS直聘/猎聘/智联招聘）的算法穿透规则和 HR 人工筛选习惯。**

核心原则：
- **拒绝千篇一律**：按岗位类型动态决定技能维度和表达重点
- **基于真实经历**：不编造数据，信息不足时主动追问
- **ATS 友好 + 平台算法友好**：关键词自然嵌入，结构清晰可解析
- **行动导向**：不只给建议，直接生成可投递的成品
- **AI 时代标准**：检查是否包含 AI 开发框架、AI 编程工具、监控运维、线上作品集等现代要素
- **平台差异化**：BOSS直聘/猎聘/智联招聘采用不同优化策略


## AI 时代简历黄金法则（2026 版）

在与传统简历相同的结构基础上，现代简历必须额外满足以下要求：

### 1. 求职意向带 AI 标签
格式：`岗位名称（具备 XX AI 经验）`
- 示例：`Java 后端开发（具备 AI Agent 全栈开发经验）`
- 原因：HR 筛简历平均 10 秒，必须在开头 3 秒抓住注意力

### 2. 作品集网站 + GitHub 必放头部
- 头部显眼位置放个人作品集 URL 和 GitHub 链接
- 10 份简历中 8 份没有线上作品地址 → 有就是区分度
- 用 AI 编程工具 1 小时就能搭出作品集网站，成本极低

### 3. AI 技能分层展示（专业技能模块）
AI 相关技能按 4 个层次组织：
- **第 1 层 — AI 开发框架**: Spring AI、LangChain4j、RAG、MCP、Agent
- **第 2 层 — AI 应用实战**: RAG 知识库构建、Prompt 优化、AI 产品设计
- **第 3 层 — AI 编程工具**: Cursor、Claude Code、GitHub Copilot、MCP Server/Skills 开发
- **第 4 层 — AI 编程模式**: Vibe Coding、SDD（规约驱动开发）、Harness Engineering

### 4. 监控告警 + SEO/GEO（后端/全栈必须）
- 监控：Prometheus、Grafana、ARMS、可观测性建设
- SEO/GEO：搜索引擎优化 + 生成式引擎优化
- 体现"写完代码之后"的能力 — 上线、监控、推广、运营

### 5. 项目必须上线，附可访问地址
- 每个项目必须有线上地址或脱敏文档
- "能在本地跑"和"能部署上线给用户用"是两个层次
- 不上线 = 面试官无法验证 → 信任度大打折扣

### 6. 每一条经历带量化数据
- 规模类：QPS、DAU、数据量、用户数
- 提升类：性能提升 X%、效率提升 X 倍、成本降低 X%
- 真实测试得来，不编造（可用 JMeter 等工具实测）

### 7. 体现自主性（独当一面）
- 不只是"参与了 XX 项目" → 要写你做了什么决策、用了什么方案
- 体现从 0 到 1 的架构设计、独立交付产品的能力
- "代码由 AI 生成"在现代是加分项 — 前提是你能把控全局


## 国内招聘平台实战规则（2026 版）

### 一、平台算法穿透（BOSS直聘/猎聘/智联招聘）

#### 1. 关键词精准匹配
- **显性要求提取**：从 JD 中提取硬性技能词（如 Java、Spring Boot、MySQL），必须 100% 覆盖
- **行业前瞻性词汇**：补充该岗位近两年新兴关键词（如 AI Agent、RAG、MCP、可观测性）
- **动态技能关键词库**：按岗位类型建立关键词库，见 `references/job_types.md`
- **关键词密度**：保持 8%-12%，实测通过平台 AI 初筛率可提升 2.9-3 倍
  - 过低（<5%）：算法识别为不匹配
  - 过高（>15%）：被判定为关键词堆砌，降权处理

#### 2. 语义化表达升级
将扁平技能词升级为"技能+场景+结果"的完整语义：

| ❌ 扁平表达 | ✅ 语义化表达 |
|------------|------------|
| Python | Python + Flask + RESTful API 优化接口响应时间 40% |
| 负责运营 | 主导用户增长项目，3 个月内实现日活提升 42% |
| Java 开发 | 基于 Spring Boot 微服务架构，支撑日均 100 万+ 订单处理 |

目的：让机器（算法）和人工（HR）双重识别你的能力。

#### 3. 格式机读友好
- 使用标准章节标题（工作经历、教育背景、技能、项目经验），不要用创意标题
- **禁止表格/图片/复杂排版**：ATS 系统解析表格错误率 >30%，图片内容完全丢失
- 避免页眉页脚存放关键信息
- 确保平台 ATS 系统毫秒级解析不丢信息

### 二、HR 6 秒黄金法则（人工筛选层）

HR 平均用 6 秒决定是否继续看一份简历。

#### 1. 页眉三要素（必须在头部第一屏）
```
[姓名] (18pt 加粗)
[目标岗位] (14pt 蓝色，与 JD 100% 一致)
[即时联系方式] (手机 + 邮箱 + 所在地)
[作品集/GitHub] (如有，放第二行)
```

#### 2. 单屏呈现原则
- **超过一屏的内容被系统截断概率高达 38%**
- 核心优势、最近 2 份工作经历、最高含金量项目必须在第一屏可见
- 排版测试：用 A4 纸打印预览，第一屏必须包含：姓名+岗位+核心优势+最近1份工作

#### 3. 信息密度控制
- 每行不超过 25 字（中文）
- 每段 3-4 行
- 确保 HR 6 秒内读完核心信息

### 三、数据驱动的内容量化（STAR + CAR 双框架）

#### STAR 法则（适合完整项目描述）
```
情境(Situation)   → 项目背景、业务规模、面临问题
任务(Task)         → 你负责的具体目标
行动(Action)        → 你采取的技术方案和决策
结果(Result)       → 量化成果（必须带数字）
```

#### CAR 法则（适合单一突破点突出）
```
挑战(Challenge)     → 遇到的具体难题或瓶颈
行动(Action)        → 你采取的解决措施
结果(Result)       → 突破性成果（必须带数字）
```

#### 量化公式（高密度句式）
```
「技术/行动 + 资产 + 指标 + 业务效果」

❌ 负责后端接口开发
✅ 基于 Spring Boot + MyBatis 重构订单状态机，履约超时率从 5% 降至 0.8%，年节省客服成本 120 万

❌ 参与数据库优化
✅ 主导慢查询治理，通过索引优化和分库分表将平均响应时间从 500ms 降至 50ms，QPS 提升 10 倍
```

#### 必须量化的场景
| 场景 | 量化维度 | 示例 |
|------|---------|------|
| 性能优化 | 响应时间、QPS、资源占用 | 响应时间从 500ms → 50ms |
| 系统规模 | 用户数、DAU、数据量 | 支撑日均 100 万+ 订单 |
| 效率提升 | 开发周期、人工成本 | 开发效率提升 30% |
| 质量改善 | Bug 率、可用性、拦截率 | 系统可用性 99.99% |
| 业务成果 | 转化率、留存率、成本 | 年节省成本 120 万 |

### 四、平台差异化适配策略

不同招聘平台，算法逻辑和 HR 使用习惯不同，需采用差异化策略：

#### BOSS 直聘
- **关键词优化**：完善"在线简历"中的所有技能关键词，保持完整
- **活跃度权重**：最近更新、经常在线者优先被推荐（建议每天上午 9-10 点刷新）
- **主动沟通**：用定制化开场白替代海投模板（"我看到贵公司在招 XX 岗位，我有 3 年 XX 经验，曾做过 XX 项目..."）
- **简历在线版**：确保在线简历和附件简历内容一致，算法会交叉验证

#### 猎聘
- **中高端定位**：突出管理经验、团队规模、行业深度
- **猎头沟通**：简历中体现"猎头可以怎么包装你"（行业资源、稀缺技能、管理半径）
- **薪资期望**：猎聘用户薪资普遍较高，可适当上浮 10-20%

#### 智联招聘
- **竞争激烈**：1 小时内可能有 300+ 份简历投递同一岗位，必须快速脱颖而出
- **AI 工具辅助**：建议使用 AI 工具在 1 分钟内完成关键词优化、格式重构、亮点提炼
- **简历标题**：在"求职意向"中明确写出核心优势（如"8 年 Java 经验 | 微服务架构 | AI Agent 开发"）

### 五、稳定性与职业连续性管理

HR 和算法都会评估候选人的稳定性。

#### 跳槽频率
- **5 年 3 跳为最佳**（既体现成长性，又不显得不稳定）
- **1 年 1 跳或 3 年 2 跳**：会被标记为稳定性差，建议在简历中补充说明（如"公司业务调整""寻求技术成长"）
- **超过 3 个月空窗期**：必须提供合理解释（进修/创业/健康/家庭），避免被直接筛掉

#### 职业曲线
- 呈现 3-5 年为周期的阶梯式成长（职位/薪资/技术深度三者至少其一持续上升）
- 避免职位/薪资倒挂（后一份工作比前一份差）
- 在简历中体现"学习成长"的轨迹（如从工程师 → 高级工程师 → 技术负责人）

### 六、个人品牌与第三方背书

让简历内容可被外部验证，大幅提升可信度。

#### GitHub/开源
- 用"技术成长折线图"替代默认贡献格子（截图放入作品集网站）
- 如果有开源项目，标注 Star 数和 Fork 数
- 如果没有开源项目，至少要有持续 Commit 记录（哪怕是小工具）

#### 技术博客/内容输出
- 展示"阅读-点赞-收藏-转化"漏斗数据
- 如果有公众号/掘金/CSDN，标注粉丝数和阅读量

#### 证书与荣誉
- 遵循"岗位相关度 + 稀缺性"原则排序
- 示例：ESG 审计岗优先列"Sustainability Accounting Certificate"
- 不知名比赛奖项，除非获奖难度大（Top 3%），否则不写

#### 社交证明
- 嵌入最高赞评论、官方转载记录、竞赛排名等外部验证
- 示例："知乎回答'如何学习 Java'获 10 万+ 阅读，200+ 点赞"

### 八、跨场景精准适配

不同背景的求职者，简历策略完全不同：

#### 应届生
- **课程设计商业化包装**：用"首屏 FCP<1.2s"等技术指标替代"课设"表述
- **权重分配**：实习 > 竞赛 > 学生组织 > 课程成绩
- **项目经历**：如果没有正式项目，用课程设计/个人作品/开源贡献替代

#### 转行者
- **可迁移技能提炼**：用户洞察、数据分析、项目管理（用"迁移技能地图"对齐目标岗位）
- **学习成长展示**：在个人优势中写"3 个月自学 XX 并通过 XX 认证"
- **不隐藏过去**：旧行业经验中提炼可迁移部分，而非完全删除

#### 管理岗
- **OKR 与业务增长关联写法**：技术动作自动映射业务 KR
- **体现"技术驱动业务"**：而非"技术自嗨"
- 示例：❌"带领 10 人团队完成微服务改造" → ✅"主导技术团队从单体向微服务转型，支撑业务 Q4 增长 35%"

## 使用场景与决策树

根据用户输入，判断进入哪种模式：

```
用户输入
  ├─ 有简历 + 有 JD → 模式 A: JD 驱动优化
  ├─ 有简历 + 无 JD → 模式 B: 简历审计与提升

## 模式 A — JD 驱动优化（最常用）

### A.1 JD 解析

拿到 JD 后，先提取以下结构化信息：

1. **岗位核心职责**（3-5 条）：公司希望你入职后做什么
2. **必须技能**（Must-have）：JD 中明确要求的硬技能
3. **加分技能**（Nice-to-have）：优先项或偏好
4. **AI 相关要求**：JD 中是否提及 AI 大模型、AI 编程工具、Agent、RAG、MCP 等关键词
5. **高频关键词**（5-10 个）：JD 中反复出现的词，必须在简历中出现
6. **软技能隐含要求**：从 JD 措辞推断（如"跨部门协作"→沟通能力）
7. **监控/运维/运营要求**：是否提及可观测性、监控告警、上线推广等
8. **目标平台**（如有）：BOSS直聘/猎聘/智联招聘，决定优化策略

```
## JD 解析结果

### 目标岗位：XXX
### 目标平台：BOSS直聘 / 猎聘 / 智联招聘（如有）
### 核心职责
1. ...
2. ...

### 必须技能关键词
Java, Spring Boot, MySQL, Redis, Kafka

### 加分技能关键词
Docker, K8s, 微服务

### AI 相关要求
☑ 有 / ☐ 无
关键词：RAG, MCP, Agent, AI 编程

### 监控/运维要求
☑ 有 / ☐ 无

### 高频关键词（必须覆盖）
高并发, 分布式, 性能优化, 系统设计, 可用性

### 平台优化策略
[根据目标平台输出对应策略，见"平台差异化适配"章节]

### A.2 简历诊断

| 头部信息 | 姓名、电话、邮箱 | 是否包含作品集 URL、GitHub | 页眉三要素是否完整 |
| 求职意向 | 岗位名称 | 是否附带 AI 经验标签 | 是否与 JD 100% 一致 |
| 专业技能 | 语言/框架/数据库 | 是否有 AI 开发框架、AI 编程工具、AI 编程模式 | 语义化表达升级 |

- 页眉三要素不完整（缺少目标岗位蓝色标注）

### ⚠️ 需要重写的条目
1. "负责系统开发" → 缺少技术栈和成果，且未语义化升级
2. "参与数据库设计" → 缺少规模和优化结果

### ✅ 优势条目（保持并进一步强化）
1. "QPS 从 500 提升至 5000" → 量化好，保留并补充业务效果
```

### A.3 执行优化

根据诊断结果，逐条重写。对每条需要修改的经历：

1. 保持原始事实不变
2. 加入缺失关键词（自然嵌入，不堆砌），确保密度 8%-12%
3. 补充量化数据（如无数据，标记 ⚠️ 让用户补充）
4. 将"做了什么"改成"做成了什么"，体现决策和方案
5. **语义化表达升级**：将扁平技能词升级为"技能+场景+结果"
6. 使用格式：**加粗概括 + 冒号 + 详细描述**
7. 确保核心信息在第一屏可见（HR 6秒法则）

改写示例：
```

改写后：主导核心业务后端服务的架构设计与迭代开发，日均处理请求 50 万+，系统可用性 99.9%

改写前：参与数据库优化
改写后：主导慢查询治理，通过索引优化和分库分表将平均响应时间从 500ms 降至 50ms

改写前：Python
改写后（语义化升级）：Python + Flask + RESTful API 优化接口响应时间 40%
```

**AI 时代改写增强**：在合适位置自然融入 AI 相关表达：
```
改写前：负责内容审核模块开发
改写后：主导 UGC 内容审核系统建设，将大模型能力融入审核流程，违规内容拦截率提升至 98%

## 简历内容质量标准

### 每一条工作经历必须满足
1. **具体动作** — 明确做了什么（设计、开发、优化、主导、推动）
2. **技术/方法** — 用了什么工具、技术、方法论
3. **量化结果** — 产生了什么可衡量的影响（用 STAR 或 CAR 框架展开）
4. **业务价值** — 对团队/公司/用户有什么价值
5. **语义化表达** — 技能词升级为"技能+场景+结果"

### 每一条项目经历必须满足（AI 时代增强）
1. **线上可访问** — 附 URL 或脱敏文档地址（最重要！）
2. **量化数据** — 性能提升、规模、效率提升等具体数字（用 CAR 框架突出突破点）
3. **技术细节** — 具体到技术方案层面，但不贴代码
4. **项目组合** — 理想搭配：AI应用开发项目 + AI编程项目 + 技术轮子项目
5. **STAR 框架** — 完整描述项目背景、你的任务、采取的行动、量化结果

### 量化挖掘规则
优先提取以下类型数据：
- **规模类**：QPS、DAU、数据量、用户数、交易额、PV
- **提升类**：性能提升 X%、效率提升 X 倍、成本降低 X%
- **时间类**：提前 X 周交付、响应时间从 A 优化到 B
- **质量类**：可用性 99.X%、Bug 率降低 X%、拦截率提升至 X%

如果用户未提供数据，主动追问【请在生成简历前追问】。

### 禁止事项
- ❌ 编造不存在的数据
- ❌ 夸大技术能力
- ❌ 使用空洞形容词（"认真负责""学习能力强"）
- ❌ 套用与经历无关的模板话术
- ❌ 一份简历对应所有岗位
- ❌ 只有传统技术、完全没有 AI 关键词（2026 年红线）
- ❌ 项目没有线上地址（面试官无法验证 = 信任度打折）
- ❌ 个人优势无证据支撑（每一条必须有具体事例或数据）
- ❌ 使用表格/图片/复杂排版（ATS 解析错误率 >30%）
- ❌ 关键词密度 <5% 或 >15%（算法降权）


## ATS 兼容性规则

### 结构规范
- 使用标准模块标题：工作经历、教育背景、技能、项目经验、个人简介
- 工作经历按时间倒序排列
- **不使用表格**（ATS 可能解析错误）
- **不使用图片、图表、图标**
- 不使用页眉页脚存放关键信息
- 作品集 URL 和 GitHub 放在头部显眼位置（ATS 可提取 URL）

### 关键词策略（AI 时代更新）
- JD 中的关键词必须自然出现在以下位置：
  - 技能模块（直接列出，AI 技能按 4 层结构组织）
  - 工作经历描述（融入具体成果，语义化表达）
  - 项目经验（技术栈标注）
- 关键词密度保持 8%-12%（每 100 字 3-5 个行业关键词）
- **2026 年必须覆盖的关键词类别**：AI 开发框架、AI 编程工具、监控运维、如有 AI 经验则在求职意向标注

### 格式要求
- 字体：中文使用微软雅黑/苹方，英文使用 Arial/Helvetica
- 字号：正文 10-11pt，标题 13-16pt
- 页边距：上下 1.5-2cm，左右 2-2.5cm
- 文件格式：优先 PDF，其次 DOCX
- 文件命名：`姓名_岗位_简历.pdf`


## 教育背景与奖项编写规则

1. **合并为一个板块** — 教育背景和奖项合在一起，节省空间
2. **排名策略** — 排名前 20% 以上才写，否则不主动暴露短板
3. **奖项排序** — 认可度高的、和岗位相关的放前面，不知名的比赛往后放或不写
4. **不堆砌奖项** — 面试官看到一堆没听过的比赛名称，会觉得在凑数
5. **证书与荣誉**：遵循"岗位相关度 + 稀缺性"原则排序


## 个人优势编写规则

1. **每一条必须有证据** — 不能说"学习能力强"，要说"独立阅读官方文档学习新技术"
2. **体现专业性和广度** — 且每条可验证
3. **有第三方背书** — GitHub/博客/证书/社交证明至少一项
4. 示例：
   - ❌ "我学习能力强，善于团队协作" — 空洞无物
   - ✅ "全栈能力强：已上线 10+ 个可访问产品" — 可验证
   - ✅ "热爱开源：GitHub 2 万+ Followers，1 万 Star" — 有数据
   - ✅ "技术输出：掘金文章 50 万+ 阅读，Top 10 年度创作者" — 第三方背书
5. **如果没有说服力的事例 → 删除此板块**，把空间留给项目经历
