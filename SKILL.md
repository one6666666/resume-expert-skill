# AI 简历专家 | 2026 AI 时代简历生成与优化

## 概述

本 Skill 提供从零生成简历、JD 驱动优化、ATS 兼容检测到面试准备的全流程能力。一次输入，直接输出可投递的专业简历（HTML + DOCX）。

**本 Skill 已适配 2026 年 AI 时代的简历标准**：不仅要写传统开发技能，更要体现 AI 应用开发、AI 编程工具使用、运维监控、SEO/GEO、以及独立交付产品的能力。

核心原则：
- **拒绝千篇一律**：按岗位类型动态决定技能维度和表达重点
- **基于真实经历**：不编造数据，信息不足时主动追问
- **ATS 友好**：关键词自然嵌入，结构清晰可解析
- **行动导向**：不只给建议，直接生成可投递的成品
- **AI 时代标准**：检查是否包含 AI 开发框架、AI 编程工具、监控运维、线上作品集等现代要素


## AI 时代简历黄金法则（2026 版）

在与传统简历相同的结构基础上，现代简历必须额外满足以下要求：

### 1. 求职意向带 AI 标签
格式：`岗位名称（具备 XX AI 经验）`
- 示例：`Java 后端开发（具备 AI Agent 全栈开发经验）`
- 原因：HR 筛简历平均 10 秒，必须在开头 3 秒抓住注意力

### 2. 作品集网站 + GitHub 必放头部
- 头部显眼位置放个人作品集 URL 和 GitHub 链接
- 10 份简历中 8 份没有线上作品地址 → 有就是区分度
- 用 AI 编程工具 1 小时就能搭出作品集网站，成本极低

### 3. AI 技能分层展示（专业技能模块）
AI 相关技能按 4 个层次组织：
- **第 1 层 — AI 开发框架**: Spring AI、LangChain4j、RAG、MCP、Agent
- **第 2 层 — AI 应用实战**: RAG 知识库构建、Prompt 优化、AI 产品设计
- **第 3 层 — AI 编程工具**: Cursor、Claude Code、GitHub Copilot、MCP Server/Skills 开发
- **第 4 层 — AI 编程模式**: Vibe Coding、SDD（规约驱动开发）、Harness Engineering

### 4. 监控告警 + SEO/GEO（后端/全栈必须）
- 监控：Prometheus、Grafana、ARMS、可观测性建设
- SEO/GEO：搜索引擎优化 + 生成式引擎优化
- 体现"写完代码之后"的能力 — 上线、监控、推广、运营

### 5. 项目必须上线，附可访问地址
- 每个项目必须有线上地址或脱敏文档
- "能在本地跑"和"能部署上线给用户用"是两个层次
- 不上线 = 面试官无法验证 → 信任度大打折扣

### 6. 每一条经历带量化数据
- 规模类：QPS、DAU、数据量、用户数
- 提升类：性能提升 X%、效率提升 X 倍、成本降低 X%
- 真实测试得来，不编造（可用 JMeter 等工具实测）

### 7. 体现自主性（独当一面）
- 不只是"参与了 XX 项目" → 要写你做了什么决策、用了什么方案
- 体现从 0 到 1 的架构设计、独立交付产品的能力
- "代码由 AI 生成"在现代是加分项 — 前提是你能把控全局


## 使用场景与决策树

根据用户输入，判断进入哪种模式：

```
用户输入
  ├─ 有简历 + 有 JD → 模式 A: JD 驱动优化
  ├─ 有简历 + 无 JD → 模式 B: 简历审计与提升
  ├─ 无简历 + 有 JD → 模式 C: 从 JD 反向生成

  └─ 只要格式转换   → 模式 F: 格式输出
```


## 模式 A — JD 驱动优化（最常用）

### A.1 JD 解析

拿到 JD 后，先提取以下结构化信息：

1. **岗位核心职责**（3-5 条）：公司希望你入职后做什么
2. **必须技能**（Must-have）：JD 中明确要求的硬技能
3. **加分技能**（Nice-to-have）：优先项或偏好
4. **AI 相关要求**：JD 中是否提及 AI 大模型、AI 编程工具、Agent、RAG、MCP 等关键词
5. **高频关键词**（5-10 个）：JD 中反复出现的词，必须在简历中出现
6. **软技能隐含要求**：从 JD 措辞推断（如"跨部门协作"→沟通能力）
7. **监控/运维/运营要求**：是否提及可观测性、监控告警、上线推广等

输出格式：
```
## JD 解析结果

### 目标岗位：XXX
### 核心职责
1. ...
2. ...

### 必须技能关键词
Java, Spring Boot, MySQL, Redis, Kafka

### 加分技能关键词
Docker, K8s, 微服务

### AI 相关要求
☑ 有 / ☐ 无
关键词：RAG, MCP, Agent, AI 编程

### 监控/运维要求
☑ 有 / ☐ 无

### 高频关键词（必须覆盖）
高并发, 分布式, 性能优化, 系统设计, 可用性
```

### A.2 简历诊断

对照 JD 解析结果，诊断现有简历。**AI 时代额外检查项**见下表：

| 检查维度 | 传统检查 | AI 时代新增检查 |
|---------|---------|---------------|
| 关键词覆盖 | JD 技术词覆盖 | AI 开发框架、AI 编程工具、监控运维词覆盖 |
| 头部信息 | 姓名、电话、邮箱 | 是否包含作品集 URL、GitHub |
| 求职意向 | 岗位名称 | 是否附带 AI 经验标签 |
| 专业技能 | 语言/框架/数据库 | 是否有 AI 开发框架、AI 编程工具、AI 编程模式 |
| 工作经历 | 职责/成果 | 是否体现架构决策、运维能力、技术前瞻性 |
| 项目经验 | 技术栈/描述 | 是否上线可访问、有量化数据、是否包含 AI 项目 |
| 整体结构 | 模块顺序 | AI 项目是否放在显眼位置 |

常规诊断输出格式：
```
## 简历诊断报告

### 📊 整体评分
- JD 关键词覆盖率：60%（15/25）
- 经历相关性：⭐⭐⭐
- 量化表达：⭐⭐
- 结构清晰度：⭐⭐⭐⭐
- AI 时代适配度：⭐⭐  ← 新增

### ❌ 缺失关键词
Docker, K8s, 微服务架构, 容灾设计

### 🔴 AI 时代缺失项
- 头部缺少作品集网站 URL
- 头部缺少 GitHub 链接
- 专业技能缺少 AI 相关技能（至少一条）
- 项目经验中无上线地址

### ⚠️ 需要重写的条目
1. "负责系统开发" → 缺少技术栈和成果
2. "参与数据库设计" → 缺少规模和优化结果

### ✅ 优势条目（保持）
1. "QPS 从 500 提升至 5000" → 量化好，保留
```

### A.3 执行优化

根据诊断结果，逐条重写。对每条需要修改的经历：

1. 保持原始事实不变
2. 加入缺失关键词（自然嵌入，不堆砌）
3. 补充量化数据（如无数据，标记 ⚠️ 让用户补充）
4. 将"做了什么"改成"做成了什么"，体现决策和方案
5. 使用格式：**加粗概括 + 冒号 + 详细描述**

改写示例：
```
改写前：负责公司后端服务开发与维护
改写后：主导核心业务后端服务的架构设计与迭代开发，日均处理请求 50万+，系统可用性 99.9%

改写前：参与数据库优化
改写后：主导慢查询治理，通过索引优化和分库分表将平均响应时间从 500ms 降至 50ms
```

**AI 时代改写增强**：在合适位置自然融入 AI 相关表达：
```
改写前：负责内容审核模块开发
改写后：主导 UGC 内容审核系统建设，将大模型能力融入审核流程，违规内容拦截率提升至 98%
```

## 模式 C — 从 JD 反向生成

用户只有 JD，没有简历。先询问关键信息，再生成。

### C.1 信息收集（必问）

1. 姓名、联系方式、所在地
2. 求职意向（可基于 JD 推导，询问是否有 AI 相关经验可加标签）
3. 作品集网站 / GitHub（如有）
4. 教育背景（学校、专业、学历、时间、排名如进前 20% 建议写）
5. 工作经历（公司、职位、时间段、主要职责和成果、产品规模数据）
6. 项目经验（项目名、你的角色、技术栈、成果、线上地址）
7. 技能清单（关注是否有 AI 开发框架、AI 编程工具）
8. AI 相关经验（用过哪些 AI 工具、做过 AI 项目、开发过 Agent/Skills）

### C.2 生成策略

根据 JD 反向定制：
- 将与 JD 最相关的经历放在最前面
- 技能模块优先展示 JD 所需技能，AI 技能按 4 层结构组织
- 项目描述中嵌入 JD 高频关键词
- 如果 JD 提到 AI/大模型 → 确保简历中有 AI 项目
- 如果 JD 提到监控运维 → 确保工作经历中有相关内容


## 模式 D — 从零生成

用户没有任何材料。用对话式收集信息：

```
请告诉我以下信息，我帮你生成简历：

1. 你应聘什么岗位？有没有目标 JD？
2. 你的教育背景？（学校、专业、学历、排名如在前20%可写）

7. 你用过哪些 AI 编程工具？（Cursor/Claude Code/Copilot 等）
```

信息不足时继续追问，直到信息充分后再生成。


## 模式 E — 面试准备

3. **弱点预判**：简历中可能被追问的薄弱点及应对
4. **反向提问建议**（3 个）：面试结束时可以反问面试官的问题
5. **AI 时代加分项准备**：如果简历中有 AI 相关项目 → 准备"你是如何用 AI 编程工具提效的"类问题

## 简历内容质量标准

### 每一条工作经历必须满足
1. **具体动作** — 明确做了什么（设计、开发、优化、主导、推动）
2. **技术/方法** — 用了什么工具、技术、方法论
3. **量化结果** — 产生了什么可衡量的影响
4. **业务价值** — 对团队/公司/用户有什么价值

### 每一条项目经历必须满足（AI 时代增强）
1. **线上可访问** — 附 URL 或脱敏文档地址（最重要！）
2. **量化数据** — 性能提升、规模、效率提升等具体数字
3. **技术细节** — 具体到技术方案层面，但不贴代码
4. **项目组合** — 理想搭配：AI应用开发项目 + AI编程项目 + 技术轮子项目

### 量化挖掘规则
优先提取以下类型数据：
- **规模类**：QPS、DAU、数据量、用户数、交易额、PV
- **提升类**：性能提升 X%、效率提升 X 倍、成本降低 X%
- **时间类**：提前 X 周交付、响应时间从 A 优化到 B
- **质量类**：可用性 99.X%、Bug 率降低 X%、拦截率提升至 X%

如果用户未提供数据，主动追问【请在生成简历前追问】。

### 禁止事项
- ❌ 编造不存在的数据
- ❌ 夸大技术能力
- ❌ 使用空洞形容词（"认真负责""学习能力强"）
- ❌ 套用与经历无关的模板话术
- ❌ 一份简历对应所有岗位
- ❌ 只有传统技术、完全没有 AI 关键词（2026 年红线）
- ❌ 项目没有线上地址（面试官无法验证 = 信任度打折）
- ❌ 个人优势无证据支撑（每一条必须有具体事例或数据）


## ATS 兼容性规则

### 结构规范
- 使用标准模块标题：工作经历、教育背景、技能、项目经验、个人简介
- 工作经历按时间倒序排列
- 不使用表格（ATS 可能解析错误）
- 不使用图片、图表、图标
- 不使用页眉页脚存放关键信息
- 作品集 URL 和 GitHub 放在头部显眼位置（ATS 可提取 URL）

### 关键词策略（AI 时代更新）
- JD 中的关键词必须自然出现在以下位置：
  - 技能模块（直接列出，AI 技能按 4 层结构组织）
  - 工作经历描述（融入具体成果）
  - 项目经验（技术栈标注）
- 不要过度堆砌（每 100 字出现 3-5 个行业关键词即可）
- **2026 年必须覆盖的关键词类别**：AI 开发框架、AI 编程工具、监控运维、如有 AI 经验则在求职意向标注

### 格式要求
- 字体：中文使用微软雅黑/苹方，英文使用 Arial/Helvetica
- 字号：正文 10-11pt，标题 13-16pt
- 页边距：上下 1.5-2cm，左右 2-2.5cm
- 文件格式：优先 PDF，其次 DOCX
- 文件命名：`姓名_岗位_简历.pdf`


## 教育背景与奖项编写规则

1. **合并为一个板块** — 教育背景和奖项合在一起，节省空间
2. **排名策略** — 排名前 20% 以上才写，否则不主动暴露短板
3. **奖项排序** — 认可度高的、和岗位相关的放前面，不知名的比赛往后放或不写
4. **不堆砌奖项** — 面试官看到一堆没听过的比赛名称，会觉得在凑数


## 个人优势编写规则

1. **每一条必须有证据** — 不能说"学习能力强"，要说"独立阅读官方文档学习新技术"
2. **体现专业性和广度** — 且每条可验证
3. 示例：
   - ❌ "我学习能力强，善于团队协作" — 空洞无物
   - ✅ "全栈能力强：已上线 10+ 个可访问产品" — 可验证
   - ✅ "热爱开源：GitHub 2 万+ Followers，1 万 Star" — 有数据
4. **如果没有说服力的事例 → 删除此板块**，把空间留给项目经历


## 工作流示例

### 示例 1：用户提供简历和 JD

```
用户：帮我根据这个 JD 优化简历
【附 JD 和简历文本】

→ 进入模式 A
→ A.1 解析 JD（含 AI 相关检测），提取关键词和职责
→ A.2 诊断简历（含 AI 时代专项检查），输出诊断报告
→ A.3 逐条重写不匹配的经历，融入 AI 关键词
→ 输出：优化后的简历全文 + 修改说明

追问：需要我生成 HTML 或 DOCX 版本吗？
```

### 示例 2：用户只需要面试建议

```
用户：我拿到了字节跳动后端开发的面试，帮我准备一下
【附 JD 和简历】

→ 进入模式 E
→ 分析 JD 和简历
→ 输出：高频考点、亮点故事线、弱点预判、反问建议
→ AI 时代加分：预判"如何用 AI 提效"类问题
```

### 示例 3：用户从零开始

```
用户：我要应聘数据分析师，帮我写一份简历

→ 进入模式 D
→ 开始信息收集对话
→ 逐项询问：教育、工作、项目、技能、AI 工具使用
→ 信息充分后，参考 references/job_types.md 生成
→ 输出 HTML + DOCX 双版本
```
