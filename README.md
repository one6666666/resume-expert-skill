### v4.0 新增功能

- 🆕 **模式 G — 多版本管理**：一岗一版，自动生成版本差异对比表
- 🆕 **模式 H — 转行者适配**：可迁移技能地图，12 种行业转岗路径
- 🆕 **模式 I — 薪资谈判准备**：市场薪资调研、谈判策略、话术生成、offer 对比
- 🆕 **模式 J — 作品集案例写作**：将简历亮点扩展为详细案例，含技术细节和验证链接
- 🆕 **贡献动词门槛规则**：支持/协助 < 参与 < 负责 < 推动 < 主导（禁止随意升档）
- 🆕 **5 维度评分体系**：JD匹配度 / ATS兼容性 / HR吸引力 / 面试准备度 / 可信度
- 🆕 **证据映射规则**：已覆盖 / 弱覆盖 / 未覆盖 / 可补充 / 不建议硬凑
- 🆕 **风险识别与叙事修复**：自动检测外包经历、玩具项目、跳槽频繁等风险信号
- 🆕 **模拟面试追问生成**：针对每条核心经历生成 2-3 个预判问题及回答要点
- 🆕 **BOSS直聘开场白生成器**：定制化沟通话术，替代海投模板
- 🆕 **ATS 100 分检查表**：结构化的自检清单
- 🆕 **新增岗位类型**：高管（C-level/VP/技术总监），共支持 12+ 岗位

## 2026 年 AI 时代简历黄金法则

本 Skill 内置以下规则，**自动执行**：

1. **求职意向带 AI 标签**：`Java 后端开发（具备 AI Agent 全栈开发经验）`
2. **作品集 + GitHub 必放头部**：10 份简历中 8 份没有 → 有就是区分度
3. **AI 技能 4 层结构**：AI 开发框架 → AI 应用实战 → AI 编程工具 → AI 编程模式
4. **监控告警 + SEO/GEO**：体现"写完代码之后"的能力
5. **项目必须上线**：附可访问地址，否则信任度打折
6. **每一条经历带量化数据**：用 STAR/CAR 框架展开
7. **体现自主性**：从 0 到 1 的架构设计、独立交付

| **特殊注意** | 在线版和附件必须一致 | 猎头可帮你包装 | 用 AI 工具 1 分钟优化 |

## 5 维度评分体系

简历诊断时，自动从以下 5 个维度评分（每项 ⭐1-5）：

| 维度 | 评分标准 |
|------|---------|
| **JD 匹配度** | 关键词覆盖率（必须技能 100% 覆盖 = 5 分） |
| **ATS 兼容性** | 格式规范、关键词密度 8%-12%、无表格图片 |
| **HR 吸引力** | 页眉三要素完整、单屏核心信息可见、信息密度合规 |
| **面试准备度** | 每条经历可展开 3 分钟、量化数据可验证 |
| **可信度** | 贡献动词与实际匹配、无夸大、团队/个人贡献区分明确 |
